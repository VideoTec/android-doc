以往的Android开发有一个头疼的且拖慢速度的问题，
就是你每改一行代码要想看到结果必须要编译运行到手机或者模拟器上，
而且需要从头（可能是登录界面）一直点击到你修改的界面为止。
开发一个完整的Android App你可能要经历无数个重复编译运行的过程，
严重的拖慢了开发进度。

最近React Native for Android可谓是解决了这个问题，
修改代码可以直接在模拟其上刷新出来当前修改的界面（毕竟是用web技术）。
于是乎Google能看得下去让FB占领自己的开发领域吗？不可能！
